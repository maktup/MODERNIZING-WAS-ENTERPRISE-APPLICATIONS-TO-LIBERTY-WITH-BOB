# OBJETIVO:
Entender como trabajar con BOB para cubrir los escenarios asociados a MODERNIZACIÓN:

_ El CASO DE USO en el cual se está enfocando esta SOLUCIÓN, es la MODERNIZACIÓN de PROYECTOS antiguos que actualmente tienen muchos clientes, que fueron hace 10 o 15 años implementados con TECNOLOGÍA WAS & que ahora son considerados como LEGACY.

_ Estas MODERNIZACIONES requieren de mucho conocimiento en PROGRAMACIÓN (en este caso JAVA) & de muchos meses de inversión de tiempo para ejecutarlos MANUALMENTE.

_ BOB entra a ayudar en este trabajo, agilizando la MODERNIZACIÓN en un 800%, minimizando los tiempos & trabajo MANUAL en gran cantidad. 

  

# RECOMENDACIONES:
Estas son algunas de las RECOMENDACIONES asocias al trabajo de MODERNIZACIÓN de aplicaciones empresariales JAVA con BOB:

_	BOB será el arma por excelencia para agilizar la MODERNIZACIÓN de las ENTERPRISE APPLICACIÓN & funcionará bastante bien, solo se requiere conocer bien como utilizarlo.

_	Si estas afrontando una MODERNIZACIÓN de ENTERPRISE APPLICACIÓN basados en JAVA, si o si quien manipule a BOB deberá ser un: ARQUITECTO o PROGRAMADOR, con muchos conocimientos de la TECNOLOGÍA a donde se desea MODERNIZAR.

_	BOB para poder trabajar bien, requerirá sí o sí que el proyecto esté MAVENIZADO, así que lo primero que se debe realiza es MAVENIZAR el PROYECTO BASE de WAS.

_	BOB utilizará como herramienta base el VSC (VISUAL STUDIO CODE), en donde se tendrá que cargar el directorio del PROYECTO que se desea MODERNIZAR. 

_	Es una BUENA PRÁCTICA que se tenga abierto el mismo PROYECTO BASE (MAVENIZADO) con otra IDE, en este caso asociada a JAVA como la de ECLIPSE.

_	BOB trabaja en base a PROMPTs & para que trabaje correctamente se requiere analizar con mucho detalle el PROMPT correcto que se utilizará desde el inicio para la MODERNIZACIÓN, ya que BOB generará un PLAN de acciones encadenadas como solución (mientras más DETALLE mejor). 

_	NO se debe pensar que solo con ingresarle un PROMTP a BOB se realizará toda la MODERNIZACIÓN, ya que es un trabajo progresivo el cual generará resultados y ante las FALLAs que vayan apareciendo se necesitará volverlas a cruzarlas con BOB hasta lograr la solución definitiva.

_	BOB analizará el PROMPT ingresado & generará un PLAN de trabajo, para el cual irá solicitando CONFIRMACIÓN manual, para cada ejecución.
