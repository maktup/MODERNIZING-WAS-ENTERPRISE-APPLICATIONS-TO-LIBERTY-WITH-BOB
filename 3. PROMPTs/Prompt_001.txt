
Actúa como un ARQUITECTO y PROGRAMADOR senior experto en:
IBM WebSphere Application Server tradicional (WAS), Open Liberty / WebSphere Liberty, Java Enterprise, Maven y migraciones empresariales.

Necesito que migres un proyecto JAVA EMPRESARIAL que actualmente tiene estructura y configuración orientada a WAS tradicional hacia un proyecto compatible con IBM LIBERTY usando MAVEN.
 
I. OBJETIVO PRINCIPAL: 
Transformar el proyecto para que:

- Tenga una estructura estándar Maven.
- Sea totalmente compatible con IBM Liberty.
- Compile correctamente con Maven (mvn clean package).
- Mantenga la lógica funcional existente.
- Tenga una estructura limpia, mantenible y moderna.
- Sea compatible con Java 11 o superior.


II. ENFOQUE DE TRABAJO: 
1. Analiza la estructura actual del proyecto.
2. Identifica elementos específicos de WAS tradicional.
3. Detecta dependencias, configuraciones y código incompatible con Liberty.
4. Propón y genera una nueva estructura basada en Maven.
5. Refactoriza el código cuando sea necesario (sin alterar lógica de negocio).
6. Corrige:
   - errores de sintaxis
   - imports obsoletos
   - APIs deprecadas
   - problemas de encoding (UTF-8)
7. Reubica clases y recursos mal estructurados.
8. Convierte configuraciones propietarias de WAS a Liberty.
9. Si existen librerías en carpetas (.lib o jars locales):
   - valida si existen en repositorios Maven
   - agrégalas al pom.xml
   - elimina los jars físicos del proyecto
10. El proyecto LIBERTY debe soportar OPENTELEMETRY.   
11. Si hay referencias a:
   - JNDI
   - datasources
   - JMS
   - security roles
   - shared libraries
   - bindings de WAS
   conviértelas a configuraciones compatibles con Liberty.
12. Si las librerias usadas están DEPRECATED actualiza la versión de las librerias.
13. Si hay dudas, prioriza una solución práctica, compilable y mantenible.
14. Si existen clases JAVA duplicadas en otros directorios por la migración, soluciona eliminando las que se requieran. 


III. ESTRUCTURA MAVEN ESPERADA:  
Estructura el proyecto JAVA LIBERTY para que tenfa:

- src/main/java
- src/main/resources
- src/main/webapp
- src/test/java


IV. MIGRACIÓN DE CONFIGURACIÓN: 
Analiza y migra elementos como:
- datasources
- JNDI
- JMS (colas, connection factories)
- variables de entorno
- shared libraries
- classloader policies
- security roles
- context roots
- propiedades del servidor
- bindings específicos de WAS

V. Incluye migración de archivos como:
- web.xml
- ibm-web-bnd.xml
- ibm-application-bnd.xml
- ibm-ejb-jar-bnd.xml
- otros descriptores WAS

Convierte todo a configuraciones compatibles con LIBERTY (server.xml y otros necesarios).


IV. ARCHIVOS A GENERAR / ACTUALIZAR: 
Genera o actualiza:

- pom.xml (completo y funcional)
- server.xml (para Liberty)
- web.xml (si aplica)
- MANIFEST.MF (si aplica)
- archivos de configuración necesarios


VII. RESULTADO ESPERADO: 
El proyecto final debe:

- Compilar correctamente con Maven
- Ejecutarse en IBM Liberty
- No depender de librerías propietarias de WAS innecesarias
- Tener dependencias correctamente gestionadas en pom.xml
- Tener configuración limpia y moderna


VIII. FORMATO DE RESPUESTA: 
Genera o actualiza un reporte estructurado con:

1. Diagnóstico del proyecto actual.
2. Lista de incompatibilidades con Liberty.
3. Nueva estructura Maven propuesta.
4. Código y archivos transformados (completos).
5. Librerías y directorios eliminados (ya no necesarios).
6. Cambios realizados para compilar en Maven.
7. Cambios realizados para desplegar en Liberty.
8. Riesgos o validaciones pendientes.

IX. IMPORTANTE:
- No des respuestas genéricas.
- Genera archivos completos y listos para usar.
- Prioriza que el proyecto compile y arranque correctamente.
